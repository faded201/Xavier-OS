```

│          │          │          │       │          │
1          2          3          4  ...  89         90
```

**Breakdown:**
- **Total duration:** 1380 seconds (23 minutes)
- **Total images:** 90 images
- **Interval:** 1380 / 90 = **15.3 seconds per image**

## 📱 USER EXPERIENCE FLOW

### **Loading Book → Playing Episode → 23-Minute Journey**

```
1. User clicks "PLAY" on a book
   └─ Story is generated
   └─ Audio TTS is generated
   └─ 🎬 Image batch generation STARTS in background

2. Audio plays first chunk
   └─ 🎬 Status shows: "Generating 90 images... 8-12 minutes"
   └─ First image might not be ready yet

3. As audio plays (~3 seconds in)
   └─ 🎬 First batch of images ready
   └─ Image #1 displays with counter "1 / 90"

4. Every ~15.3 seconds
   └─ Audio continues playing
   └─ Next image auto-advances
   └─ Counter updates to "2 / 90", "3 / 90", etc.
   └─ Progress bar fills with each image

5. At 23:00 (end of audio)
   └─ 🎬 Image #90 displays with counter "90 / 90"
   └─ Could have up to 90 different cinematic scenes watched

6. User can:
   └─ Replay episode (see same image sequence)
   └─ Move to next episode (new 90 images generated)
   └─ Switch to different book (different 90 images)
```

## ⚙️ TECHNICAL DETAILS

### **Image Generation**

- **API:** Pollinations.ai (free, no limits)
- **Resolution:** 1920×1080 (Full HD)
- **Format:** JPEG
- **Size:** ~50-100KB per image
- **Total data:** ~4.5-9 MB per episode
- **Generation time:** 8-12 minutes
- **Generation method:** Parallel batches of 5 (rate-limited)

### **Image Delivery**

- **Caching:** Images cached after generation
- **Sync:** Real-time sync with audio playback
- **Storage:** Client-side (localStorage) or server
- **Streaming:** Progressive loading (doesn't wait for all 90)

### **Memory Integration**

- **Source:** Character sheet from story
- **Update frequency:** Per episode
- **Persistence:** Episode-specific (resets per episode)
- **Injection points:** 5x per prompt (character, world, story, relationships, visual)
